Getting Started

This guide walks you through adding Blox Material to new or existing Angular projects. It assumes the use of Angular CLI for creating and maintaining your project. The guide also assumes that you are running at least node 6.9.x and npm 3.x.x, which you can verify by running node -v and npm -v. Please refer to Node Downloads for instructions to install node and npm on your computer. To install Angular CLI, run:

This will install the ng command globally on your system. If Angular CLI is already installed on your system, make sure you have at least version 1.5.2 installed, by running ng -v. To upgrade to the latest version just run the install command shown above again.

(You can find that under apps/styles in the file). Since scss is an extension of css, this will not affect your existing styles.

For existing projects, also make sure that you are using at least Angular CLI 1.5.2 (Angular CLI uses the version installed for your project, not the globally installed version). This can be checked by running ng -v in the project directory. If the project uses an older Angular CLI version, upgrade it by running npm install --save-dev @angular/cli in your project directory.

The example also includes the @angular/forms FormsModule. Using the FormsModule is not a requirement of Blox Material. All Blox Material components can also be used without the FormsModule. However, the FormsModule makes building forms much easier, with easy binding of controls to data, addition of validation rules, display of errors, and much more. Please refer to Angular Forms Guide for more in depth information.

Blox Material is fully compatible with @angular/forms, and supports both the FormsModule, and the ReactiveFormsModule. But all Blox Material features can also be used without the @angular/forms package.

Step 5: Compatibility with older browsers

The web app you just created will not work with older browsers, such as IE11. Blox Material depends on Google's Material Components for the Web, which is distributed in different @material/* npm packages. These packages all use es2015 (also called es6, or ecmascript 2015) features that not all browsers support.
